# MahjongTing

> 面向移动端实时场景的麻将牌视觉识别与听牌辅助系统。  
> 当前版本支持 **广东麻将 / 四川麻将** 两种规则模式，集成 **图形化录入、听牌/胡牌计算、iOS 相机扫描、训练数据采集与导出**。

---

## 项目简介

本项目是一个基于 **SwiftUI** 开发的 iOS 麻将辅助应用，目标是在移动端完成从牌面录入到规则计算、再到视觉识别的数据闭环。

当前仓库重点包括：

- **手牌图形化录入**：支持万 / 筒 / 索 / 字牌的点选输入
- **多规则支持**：支持广东麻将与四川麻将
- **听牌 / 胡牌计算**：支持 13 张听牌计算、14 张出牌建议与胡牌判定
- **相机扫描入口**：集成基于 ARKit 的扫描界面
- **端上数据采集**：支持将扫描到的单牌图像保存为训练样本
- **开发者导出功能**：支持将本地采集的数据集导出，用于后续模型训练与实验分析

这个项目既可以作为一个实用型麻将辅助工具，也可以继续扩展为围绕 **移动端麻将视觉识别** 的研究型仓库。

## 当前功能

### 1. 麻将规则计算

- 支持 **广东麻将 / 四川麻将**
- 支持 **自动规则识别**
- 支持 **四川定缺**
- 支持 **七对** 开关
- 支持 **十三幺**（仅广东）开关
- 支持：
  - **13 张手牌**：计算听哪些牌
  - **14 张手牌**：计算“打哪张 → 听哪些牌”
  - 当前是否已经成胡的判定

> 当前版本主要做 **胡牌形判定**，暂不涉及番种计算、起胡番数限制、计分系统等完整规则扩展。

### 2. iOS 原生交互界面

- 使用 **SwiftUI** 实现原生界面
- 提供牌面点选输入、撤销、清空、设置面板等交互
- 提供规则切换、扫描开关、震动反馈等配置项

### 3. 相机扫描与识别入口

- 基于 **ARKit** 构建扫描界面
- 支持相机权限检查与会话管理
- 支持短时间 burst 抓帧
- 支持深度增强选项（设备支持时可开启 Scene Depth / Smoothed Scene Depth）

### 4. 数据集采集与导出

- 支持将单牌 patch 保存到：

```text
Documents/MahjongDataset/Training/0..33/
```

- 当前类别采用 **34 类标准麻将牌编码**
- 支持导出整个本地数据集目录
- 支持导出最新单张样本用于链路验证
- 导出时会自动生成：
  - `EXPORT_MANIFEST.txt`
  - `EXPORT_OK.txt`

## 技术栈

- **SwiftUI**：应用界面与状态驱动
- **ARKit / RealityKit**：相机扫描与 AR 会话管理
- **CoreImage / UIKit**：图像处理与 patch 保存
- **Vision / CoreML**：麻将牌分类模型接入接口
- **本地规则引擎**：胡牌、听牌、出牌建议计算

## 识别模块当前状态

当前项目已经预留了 **CoreML 识别器接口**，并封装了 `VisionCoreMLTileRecognizer`。  
扫描、抓帧、单牌 patch 提取、样本保存与导出链路已经具备；自动识别效果则取决于你接入的模型权重、类别映射以及训练数据质量。

因此，这个仓库当前更准确的描述是：

> 一个已经完成移动端交互闭环，并正在推进视觉识别模型接入与验证的麻将辅助系统。

## 模型来源与训练流程

### 当前仓库状态

本仓库当前以 **iOS 端应用、规则引擎、相机扫描与数据采集链路** 为主。  
仓库中默认 **不保证附带最终公开可复现的训练权重**；如果后续发布训练权重，建议同步公开：

- 使用的数据来源与版本
- 训练脚本与训练参数
- 类别映射表
- 权重文件来源与转换流程
- 训练日期 / commit hash / 实验记录

### 建议的研究型训练流程

1. **端上采集单牌 patch**
2. **人工检查与清洗错误样本**
3. **按类别整理为 34 类数据集**
4. **在 Python 侧进行训练 / 验证 / 导出**
5. **将模型转换为 CoreML**
6. **在 iOS 端接入推理并做类别一致性校验**
7. **回收错误案例，继续增量优化数据集**

### 可扩展的模型路线

- **路线 A：单牌分类**
  - 输入：单张牌 patch
  - 优点：链路清晰、训练门槛较低、适合当前项目阶段
- **路线 B：整图目标检测**
  - 输入：真实桌面/手牌照片
  - 优点：更接近自然交互场景，可直接从图像定位多张牌
- **路线 C：检测 + 分类 / 检测端到端**
  - 适合后续论文或更完整的研究实现

## 如何运行

### 环境要求

- Xcode
- iOS 真机（建议）
- 相机权限
- 若需要深度增强，设备需支持对应 ARKit 深度语义

### 运行步骤

1. 用 Xcode 打开项目
2. 选择真机设备运行
3. 允许相机权限
4. 进入主界面后：
   - 可手动点牌进行听牌/胡牌计算
   - 可进入扫描页进行图像采集或识别测试
