## Model Weight Notes

- No pretrained weights are bundled in this repository
- The iOS side already exposes `VisionCoreMLTileRecognizer`; once weights are added, the CoreML recognition path can be enabled
- Any future weight file added to the repository should state the training data source, label version, training date, export method, and license

## Cleaning / Annotation / Refactoring Record

### Labels and Classes
- The current single-tile classifier uses 34 classes
- The honor-tile order is fixed as: East, South, West, North, White, Green, Red
- See `CATEGORY_MAPPING_EN.md` for the mapping table

### Data Organization
- The local collection path is `Documents/MahjongDataset/Training/0..33`
- Export keeps the class directory structure and writes `EXPORT_MANIFEST.txt`
- Collected images are stored as JPEG files

### Image Processing
- Single-tile patches are generated by rectangle detection and perspective correction
- The scan page supports burst capture and leaves room for multi-frame voting

### Code Structure
- Scan, recognition, rule engine, dataset storage, and export are separated
- The recognizer is abstracted through `TileRecognizerProtocol`
- The current main entry is the native iOS UI in `NativeMahjongView.swift`
